## Write Rule

Command `0x33` output PEQ write payload:

`00 01 0A 33 <output_channel> <peq_band> <gain_lo> <gain_hi> <frequency_lo> <frequency_hi> <q_raw> <filter_type> <bypass>`

Output channel byte is `0x04 + output_index_zero_based`. PEQ band byte is `0x00` for PEQ1 through `0x08` for PEQ9.

## Read Rule

Output PEQ values are six-byte records:

`gain_u16le frequency_u16le q_raw_u8 filter_type_u8`

Formula:

`absolute_config_offset = 568 + (output_index_zero_based * 108) + 36 + (peq_band_zero_based * 6)`

Read block location:

`block = floor(absolute_config_offset / 50)`

`payload_offset = 5 + (absolute_config_offset % 50)`

For each record start, field offsets are:

- `+0`: gain u16le
- `+2`: frequency u16le
- `+4`: Q raw u8
- `+5`: filter type u8

If the offset crosses the end of a read block, continue in the next block payload at offset `0x05`.

## Bypass Masks

Output PEQ bypass is stored separately from the value records:

| Output | Status | Low Byte PEQ1..PEQ8 | High Byte PEQ9 |
| --- | --- | --- | --- |
| Out1 | observed | `0x1C:0x31` | `0x1C:0x32` |
| Out2 | derived | `0x1C:0x33` | `0x1C:0x34` |
| Out3 | derived | `0x1C:0x35` | `0x1C:0x36` |
| Out4 | derived | `0x1D:0x05` | `0x1D:0x06` |
| Out5 | derived | `0x1D:0x07` | `0x1D:0x08` |
| Out6 | derived | `0x1D:0x09` | `0x1D:0x0A` |
| Out7 | derived | `0x1D:0x0B` | `0x1D:0x0C` |
| Out8 | observed | `0x1D:0x0D` | `0x1D:0x0E` |

Low byte bit mapping: bit0 PEQ1, bit1 PEQ2, bit2 PEQ3, bit3 PEQ4, bit4 PEQ5, bit5 PEQ6, bit6 PEQ7, bit7 PEQ8.

High byte bit mapping: bit0 PEQ9.
